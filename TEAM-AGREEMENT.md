# Team Agreement — Damascene Art Project

## Communication Channels
- **Project Management:** GitHub Issues & Project Board
- **Code Reviews:** GitHub Pull Requests
- **Quick Communication:** Telegram / WhatsApp (team internal)
- **Weekly Sync:** Once per week (time to be decided)

## Working Model
- All development work must be tracked using GitHub Issues.
- No work is considered valid without an associated Issue.
- Each feature or task must include documentation.

## Responsibilities

### Project Lead
- Overall planning and coordination
- Task assignment and prioritization
- Reviewing Pull Requests
- Merging changes into `main`
- Ensuring documentation completeness

### Backend Developer
- Implement backend features using Laravel
- Design and maintain RESTful APIs
- Document backend logic and endpoints

### AI Engineers
- Develop AI services and models
- Integrate AI functionality with backend APIs
- Document AI pipelines and usage

### Frontend Developer (Future)
- Implement UI/UX
- Integrate frontend with backend APIs
- Ensure responsive and clean design

## Git Workflow Rules
- `main` branch represents the stable version.
- `develop` branch is used for active development.
- No direct push is allowed to `main`.
- All changes must go through Pull Requests.
- Pull Requests must follow the PR Template.

## Definition of Done
A task is considered "Done" when:
- Code is implemented
- Documentation is written
- Pull Request is reviewed and merged
- Issue is moved to Done column

## Conflict Resolution
- Technical disagreements are discussed in Issue comments.
- Final decision is made by the Project Lead after discussion.
- Respectful communication is mandatory.

## Commitment
All team members agree to follow this agreement throughout the project.
